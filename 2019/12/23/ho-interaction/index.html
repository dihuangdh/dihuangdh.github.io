Hand and Object Pose Estimation and Reconstruction

In 2019, there are two new papers introducing hand-object interaction, one paper (H+O: Unified Egocentric Recognition of 3D Hand-Object Poses and Interactions) estiamtes poses of hand and object, and uses hand-object relationship to do high-level tasks like action recognition. Another paper (Learning joint reconstruction of hands and manipulated objects) does jointly shape reconstruction for hand and object, and use relationships between them to help realistic reconstruction. This post give a simple introduction towards those two papers, and give some thinkings.

When we talk about Hand-Object Interaction

Before we start introducing these two paper, we should make some defination for hand-object interaction, and understand why we need it, why it is important for both researcher and engineer. When we talk about hand-object interaction, we are saying tasks which contains more information than hand-only or object-only tasks. If we are handing hands and object at the same time, there are real word constrains and high-level releationships like action between hands and object. Generally, hand-object interaction is one kind of relationship between human hands and object, including geometry constrains like contact and recognition relationship like action purpose.

Why we need to model this kind of relationship? I think it benifits both recognition and construction tasks. Here I copy part of the papers:

Human behavior can be characterized by the individual actions a person takes in interaction with the surrounding objects and the environment.

Accurate estimation of human hands, as well as their interactions with the physical world, is vital to better understand human actions and interactions. In particular, recovering the 3D shape of a hand is key to many applications including virtual and augmented reality, human-computer interaction, action recognition and imitation-based learning of robotic skills.

Pipelines

Jointly Hand-Object Pose Estimation

  • H+O: Unified Egocentric Recognition of 3D Hand-Object Poses and Interactions, CVPR19 oral
  • Author: Bugra Tekin, who is working on human pose estimation, object pose estimation. Microsoft
  • Dataset:
    • FPHA-HO. Subset of FPHA. Action recognition dataset. Contains annotations for 6D object pose and hand pose. 4 objects. 10 different action categories.
    • EgoDexter: Real dataset. 3D hand pose nad 6D object pose. Add annotation by hands. For evaluation.
    • SynthHands: Synthetic hands pose estimation dataset.
  • Pipeline: We propose a unified framework for recognizing 3D hand and object interactions by simultaneously solving four tasks in a feed-forward pass through a neural network: 3D hand pose estimation, object pose estimation, object recognition and activity classification. Our method operates on monocular color images and relies on joint features that are shared among all the tasks.
  • Hand-Object Interaction Modeling: Our scheme relies on a common output representation for both hands and objects that parametrizes their pose with 3D control points.

Jointly Hand-Object Reconstruction

  • Learning joint reconstruction of hands and manipulated objects, CVPR19

  • Author: Yana Hasson, phd student INRIA. Her first paper. Interestingly, I find I ran her I3D code before.

  • Dataset:

    • FPHA-HO. Subset of FPHA. Action recognition dataset. Contains annotations for 6D object pose and hand pose. 4 objects. 10 different action categories.
    • HIC. Hand and object meshes are provided.
    • ObMan: Synthetic. To overcome the lack of adequate training data.

  • Pipeline: Our network architecture is split into two branches. The first branch reconstructs the object shape in a normalized coordinate space. The second branch predicts the hand mesh as well as the information necessary to transfer the object to the hand-relative coordinate system. Each branch has a ResNet18 encoder pre-trained on ImageNet.

  • Hand-Object Interaction Modeling: prior knowledge that objects can not interpenetrate each other and that, when grasping objects, contacts occur at the surface between the object and the hand.

    • Repulsion: penalizes hand and object interpenetration.
    • Attraction: penalize cases in which hand vertices are in the vicinity of the object but the surfaces are not in contact.

What’s the next step?

I believe 2019 is the start year for jointly hand-object interaction learning from RGB data. Before these two papers, research on hand-object interaction are rely on RGB-D data. I haven’t read those papers yet, so I won’t compare them here. Let’s have a close look at these two papers.

For H+O, its h-o modeling is rough. But since its final task is action recognition, some loss on geometry space is acceptable. Using the same presentation, the network can predict hand key points and object keypoints at the same time, and finally get hand pose and object pose by solving a PnP problem. This method doesn’t explicitly model the interaction between hand and object, but let network to learn the relationship. To be honest, this seems a little too simple. I am not quite sure if a more accurate modeling will help recognition tasks.

Learning joint reconstruction of hands and manipulated objects uses and end-to-end pipeline to do hand-object reconstruction jointly. Make interaction as a loss item is a simple but effect idea. I believe this simple pipeline will be an important reference for next year works.

For the future trends, possible ideas are as follows:

  • Third-Person View. Nowadays hand-object are mainly working on first-person images. This is mainly because the lack of third-person view data. But we need third-person view to help understand hand-object interaction. Those will definitely help third-person view human reconstruction.
  • Add Semantic Prior. Add semantic prior will help Han-object reconstruction. Since those semantic information gives a rough idea about the relation of hand and object.
  • More Accurate Interaction Modeling. Contact is the main interaction form. So there are questions:
      1. Is contact the only interaction relationship between hand and object?
      1. How to model the contact? Are repulsion and attraction enough to describe contact?
