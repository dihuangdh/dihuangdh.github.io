Image-Based Localization

Visual localization: Given a scene representation computed from a database of geo-tagged images, e.g., a 3D model recovered via Structure-from-Motion, visual localization approaches aim to estimate the full 6 Degree-of-Freedom (6DOF) pose of a query image, i.e., the position and orientation from which the image was taken. It’s a fundamental step in many Computer Vision applications, including robotics, autonomous vehicles (self-driving cars), Augmented / Mixed / Virtual Reality, loop closure detection in SLAM, and Structure-from-Motion.

Overview

Often mothods are devided into two categories: feature-based localization or learning-based localization.

Feature-based Visual Localization

Assuming that the scene is represented by a 3D Structure-from-Motion model, the full 6DOF pose of a query image can be estimated very precisely. Approaches for feature-based visual localization compute the pose from 2D-3D matches between 2D features in the query image and 3D points in the model. These correspondences are determined by matching the descriptors of the query features against descriptors associated with the 3D points.

Learning-based Visual Localization

Compared to feature-based methods, which use an explicit representation of the scene in the form of a 3D model, these approaches implicitly represent the scene via the weights of their neural networks.

The first line of research asks whether the full, classical localization pipeline can be replaced by machine learning techniques. Such approaches regress the 6D camera pose directly from an input image using a neural network.

The second line of research keeps the overall strategy of classical visual localization, and replaces only parts with learned alternatives. In particular, methods based on scene coordinate regression learn to predict a 3D point coordinate densely, for each pixel in an image. Standard pose estimation strategies (covered in the previous parts of the tutorial) are then used to compute the camera pose from the resulting set of 2D-3D matches.

Feature-based Visual Localization

There are three most frequently talked feature-based localization methods:

  • A simple baseline
    • [ICCV 2011] Fast Image-Based Localization using Direct 2D-to-3D Matching.
    • Excellent localization effectiveness
    • Slow !
  • Vocabulary-based prioritized search (VPS)
    • [ICCV 2011] Fast Image-Based Localization using Direct 2D-to-3D Matching.
    • Idea: not all matches required for accurate localization.
    • 10x faster than simple baseline kd-tree
    • Less effective than kd-tree !
  • Active Search
    • [PAMI 2017] Efficient & Effective Prioritized Matching for Large-Scale Image-Based Localization
    • Idea: exploit co-occurrence of matches to recover matches
    • Reduce quantization artifacts.
    • Reuse existing data structures.
    • Same prioritization function.

There are some most frequently talked scene coordinate regression localization methods:

  • CNN-based

    • Large boost in accuracy: CNN learns to focus on stable structures.

Feature-based vs. Learning-based Localization

Accuracy

  • Learning Absolute Pose Regression has low accuracy due to
    • Needing to learn base poses from limited training data.
    • Needing to extrapolate at test time.
    • Using a global receptive field.
  • Learned Scene Coordinate Regression bests feature-based methods
    • On small-scale datasets.
  • Open Problems:
    • What are the limits of our datasets?
    • Robustness when changing conditions.
    • Uncertainty of pose estimation.

Scalability

  • Open Problems:
    • Feature-based methods still superior for large scale re-localization.
    • CNN-based features already outperform hand-crafted features.

Requiring an Explicit 3D Model

  • Feature-based methods require an explicit 3D reconstruction of the scene.
    • Usually done using off-the-shelf structure-from-motion tools.
  • CNN-based method encode the scene in the network weights.
    • Do we need an explicit 3D reconstruction for training?
  • Open Problems:
    • Multi-view constraints did not help much so far.
    • We still require ground truth poses.
    • Move toward SfM and SLAM.

Model Size

  • Learning-based methods encode the scene geometry into network weights.
  • The trade-off between accuracy and compression turns out to be rather good.
  • Open Problems:
    • Model size is defined by the user, not the dataset.
    • Work on network compression should be useful here.

Long-Term Localization

World is not static, appearance and geometry change.

Datasets

  • Day-Night: Aachen Day-Night Dataset
  • Seasons: CMU Seasons Dataset, RobotCar Seasons Dataset
  • Scape-Imperial Localization Dataset
  • InLoc Dataset

Learned Features

  • Higher-Level Features: Model higher-lever structures rather than low-level information (gradients, …)
  • Detector often a bottleneck under challenging conditions.
  • CNN layers define higher-level descriptors and detectors.
  • Asymmetric matching avoids storing dense descriptors
  • Joint detection & description helps handle:
    • Strong changes in scene appearance & geometry
    • Low-level vs. higher-level structures
