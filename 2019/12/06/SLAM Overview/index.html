Rough Overview of SLAM

This post is aimed for introducing basic SLAM knowledge, especially visual SLAM. Only the general pipeline will be list here, I won’t touch lots of algorithm details. This will be suitable for those who want to get rough overview of the SLAM.

This post hasn’t been finished!!! In the future, I will write more posts about the SLAM components.

What is SLAM

Simultaneous localization and mapping(SLAM) consists in the concurrent construction of a model of the environment(the map), and the estimation of the state of the robot moving within it.

– Past, Present, and Future of Simultaneous Localization and Mapping: Toward the Robust-Perception Age

To be easy, SLAM is a system which does Mapping(building a map) and Localization(locating the sensor) at the same time.

Why we need SLAM

First of all, we observe that the SLAM research done over the last decade has itself produced the visual-inertial odometry algorithm that currently represent the state-of-the-art. More generally, SLAM has directly led to the study of sensor fusion under more challenging setups (i.e., no GPS, low quality sensors) than previously considered in other literature (e.g., inertial navigation in aerospace engineering).

The second answer regards the true topology of the environment.

The third answer is that SLAM is needed for many applications that, either implicitly or explicitly, do require a globally consistent map.

– Past, Present, and Future of Simultaneous Localization and Mapping: Toward the Robust-Perception Age

Types of SLAM Camera

For visual SLAM, there are three popular types of camera: Monocular Camera, Stereo Camera, RGB-D Camera. Although nowadays new types of camera have shown up, they havn’t been widely used.

  • Monocular: Since monocular cameras are really cheap and easy to get, researchers pay lots of attention on the monocular SLAM. The main problem of Monocular SLAM is scale ambiguity.
  • Stereo Camera: Use two cameras to recover the image depth. The Baseline(distance between two cameras) need to be known. When the baseline distance is larger, the stereo can see further. The main problem of stereo camera is difficult to calibration and high computation cost.
  • RGB-D Camera: This camera type use structured light or ToF(Time-of-Flight) to get the depth map.Different from the stereo camera, RGB-D cameras don’t need to do lots of depth computation. The main problem of RGB-D camera is noisy data, limited field of view … RGB-D cameras are often used in indoor SLAM systems. For in-the-wild SLAM, it’s much more difficult to use RGB-D.

Typical SLAM Pipeline

The architecture of a SLAM system includes two main components: the front end and the back end. The front end abstracts sensor data into models that are amenable for estimation, while the back end performs inference on the abstracted data produced by the front end.

– Past, Present, and Future of Simultaneous Localization and Mapping: Toward the Robust-Perception Age

The figure from paper shows the main components of SLAM pipeline. Though clear enough, it doesn’t explain enough details. Actually, I found people prefer to use following components to describe typical SLAM pipeline:

  • Sensor Data Reading.
  • Front End: Visual Odometry. Calculate camera motion between frames, build local map.
  • Back End: Optimization. Accept camera pose of visual odometer, infomation of loop closing, then start optimization, to get global trajectory and map.
  • Loop Closure. Detect loop. If exist, pass information to back end.
  • Mapping. Build map.

Visual Odometry

Visual odeometry is the front-end of a SLAM system. It use adjacent images to estimate rough camera motion, and provide data for back-end. Basicly, there are two kinds of methods.

  • Feature Based Methods. Feature based methods need to extract features from images first, and use those features to do computations. Features based methods are always viewed as the mainstream of visual odometry, because they are stable, and insensitive to light and motion.
  • Direct Methods. Though feature is useful and robust, those methods suffer from the time consuming and feature losing. Direct methods could use all information in the images. Nowadays, direct methods are becoming important role in the SLAM area.

Let’s have a close look at the feature based methods.

  • Feature Extraction. Feature = key points + descriptor

    For SLAM, the feature should be robust, accurate and fast. ORB features are commonly used features. ORB features have scale and rotation invariance, and really fast.

  • Epipolar Constraint. Epipolar constraint is basic geometry relationship in SLAM, and can be used to calculate camera motion.

  • Triangulation. After using epipolar constrain t o get camera pose, we can use triangulation to fetch 3d points location.

  • PnP Problem. When we already get 3D points locations, and we get 2D points, we can solve PnP problem to get camera locations.

  • Point Cloud Matching: ICP.

Back End

Visual odometry estimate short-term motion and structure. However, with the error accumulation, the map will gradully become unreliable. So, SLAM need to solve a long-term optimization problem to build large scale map.

  • Extended Kalman Filter, EKF. With markov assumption, filter based methods like EKF use former state to predict next state.
  • Bundle Adjustment. Current state is related to all the states before. Nonlinear optimization based methods like BA optimize all the states.
  • Pose Graph. BA require high computaion. Pose graph give up landmard optimization, only optimize camera trajectory.
  • Factor Graph.

Loop Closure

Loop Closure add constraint for similar images, which helps to reduce error accumulation. The key is how to detect the loop.

  • Bag-of-Words, BoW. BoW use word/dictionary to describe images. Using those words, we can calculate similarity between images.
  • Similarity Calculation.
